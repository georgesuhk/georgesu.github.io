Ideal gas simulation and optimisation

Practice

Ideal gas simulated as a collection of small hard spheres undergoing perfectly elastic collisions. Thermodynamic variables are obtained and plotting again energy to test the ideal gas theory in different limits, the histogram of speeds of the particles are also obtained to compare to the expected Maxwell-Boltzmann distribution. Finally the program is also optimised to increase efficiency.

The core simulation

This project is a practice of object-oriented programming with Python for me. The program contained 3 main classes:

  • Ball (A spherical particle)
  • Container
  • Simulation

And the main functions/operations are:

  • Colliding
  • Moving
  • Obtaining time to next collision

The core loop of the simulation is as following: Finding the next collision that occurs in the system, performing the collision between the relevant particles and recording the parameters that have been altered.

Optimisations

The core simulation mentioned above is then subjected to a range of optimisations. Rather than generating a matrix for all the collisions each time a particle is moved, the collision matrix is instead only created once. The matrix is sorted in terms of ascending order of "time to collision" and the first element is taken as the next collision to perform.

Instead of recreation, we update the collision matrix for the 2 particles that have just collided. This modification is done by deleting the old collision information, calculating new time_to_collision between the 2 particles that have their trajectory modified and the other particles in the system. The new information that have been updated is inserted into the collision matrix through term by term comparison, which is more efficient than np.min() or np.sort(), as it takes advantage of the collision matrix being already ordered.
